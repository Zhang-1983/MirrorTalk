#  (MirrorTalk) 系统框架与实施路径

## 一、产品概述

**镜心 (MirrorTalk)** 是一款专门针对企业中层/新晋管理者的培训与评估产品，解决"管理者不会带团队"的核心痛点。

### 核心定位
- **目标用户**：科技企业/互联网公司的初中级管理者（特别是技术转管理的带兵人）
- **核心功能**：基于大模型的虚拟人角色扮演游戏（RPG）/ 对话式训练智能体
- **产品形态**：管理者说，AI扮演各种性格的员工进行真实反应（沉浸式实战沙盘）

---

## 二、理论依据

### 1. 彼得原理 (The Peter Principle)
员工会被晋升到他们不胜任的层级。用以说明技术骨干不等于优秀管理者，亟需管理技能培训。

### 2. 大卫·库伯的经验学习理论 (Experiential Learning Theory)
学习是一个通过"具体经验 -> 反思观察 -> 抽象概念 -> 主动实践"的循环过程。传统的PPT培训无效，必须通过"模拟实战"才能转化为能力。

### 3. 心理安全感 (Psychological Safety)
在现实中拿真人下属练手代价太大，而在AI虚拟沙盘中，管理者拥有绝对的安全感，允许试错。

```typescript
enum UserRole {
```

#### 3.2.2 剧本模块 (Scenario Module)

```typescript
// 剧本难度等级
enum DifficultyLevel {
  BEGINNER = 'beginner',      // 初级
  INTERMEDIATE = 'intermediate', // 中级
  ADVANCED = 'advanced',      // 高级

// 剧本分类
enum ScenarioCategory {
  PERFORMANCE = 'performance',    // 绩效面谈
  TERMINATION = 'termination',    // 辞退员工

// 剧本实体
interface Scenario {
  id: string;
  title: string;
  description: string;
  category: ScenarioCategory;
  difficulty: DifficultyLevel;

  background: {
    context: string;           // 场景背景
    employeeProfile: EmployeeProfile;  // 员工画像

  // 评估维度
  evaluationDimensions: EvaluationDimension[];
  
  // 预估时长（分钟）
  estimatedDuration: number;
  
  createdAt: Date;
  updatedAt: Date;
}

// 员工画像
interface EmployeeProfile {
  name: string;
  avatar: string;
  position: string;
  tenure: string;            // 司龄
  personality: string[];     // 性格标签
  strengths: string[];       // 优点
  weaknesses: string[];      // 缺点
```

#### 3.2.3 对话模块 (Dialogue Module)

```typescript
// 对话会话
interface DialogueSession {
  id: string;
  userId: string;
  scenarioId: string;
  status: DialogueStatus;
  
  // 对话历史
  messages: Message[];
  
  // 会话元数据
  metadata: {
    startTime: Date;
    endTime?: Date;
    duration?: number;       // 对话时长（秒）
    messageCount: number;    // 消息数量
    userMessageCount: number; // 用户消息数
    aiMessageCount: number;  // AI消息数
  };
  
  // 实时评估数据
  realtimeMetrics: RealtimeMetrics;
}

// 消息类型
interface Message {
  id: string;
  sessionId: string;
  role: 'user' | 'assistant' | 'system';
  content: string;
  type: 'text' | 'voice';    // 文本或语音
  timestamp: Date;
  
  // 情感分析结果（AI消息）
  emotionAnalysis?: EmotionAnalysis;
  
  // 语音转文字（如果是语音输入）
  voiceTranscript?: string;
}

// 对话状态
enum DialogueStatus {
  ACTIVE = 'active',        // 进行中
  PAUSED = 'paused',        // 暂停
  COMPLETED = 'completed',  // 已完成
  ABANDONED = 'abandoned',  // 已放弃
}

// 实时指标
interface RealtimeMetrics {
  userTalkRatio: number;     // 用户说话比例
  empathyScore: number;      // 同理心得分
  questionCount: number;     // 提问次数
  interruptCount: number;    // 打断次数
}
```

#### 3.2.4 评估模块 (Evaluation Module)

```typescript
// 评估报告
interface EvaluationReport {
  id: string;
  sessionId: string;
  userId: string;
  scenarioId: string;
  generatedAt: Date;
  
  // 综合评分
  overallScore: number;
  
  // 雷达图维度评分
  dimensionScores: DimensionScore[];
  
  // 详细分析
  analysis: {
    strengths: string[];     // 优势
    weaknesses: string[];    // 不足
    criticalMoments: CriticalMoment[]; // 关键对话节点
  };
  
  // 词云数据
  wordCloud: WordCloudItem[];
  
  // 改进建议
  suggestions: Suggestion[];
  
  // 对比数据（与历史平均/同职级对比）
  comparison?: ComparisonData;
}

// 评估维度
interface EvaluationDimension {
  id: string;
  name: string;
  description: string;
  weight: number;            // 权重
}

// 维度得分
interface DimensionScore {
  dimensionId: string;
  score: number;             // 0-100
  feedback: string;          // 该维度评价
  evidence: string[];        // 证据（引用对话内容）
}

// 关键对话节点
interface CriticalMoment {
  timestamp: Date;
  messageIndex: number;
  description: string;
  impact: 'positive' | 'negative' | 'neutral';
  suggestion: string;
}

// 词云项
interface WordCloudItem {
  word: string;
  count: number;
  sentiment: 'positive' | 'negative' | 'neutral';
  category: 'empathy' | 'blame' | 'question' | 'other';
}

// 改进建议
interface Suggestion {
  category: string;
  title: string;
  description: string;
  example: {
    before: string;          // 原话
    after: string;           // 改进后
  };
  priority: 'high' | 'medium' | 'low';
}
```

### 4.2 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    name TEXT NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('manager', 'hr', 'admin')),
    department TEXT,
    level TEXT CHECK (level IN ('junior', 'middle', 'senior')),
    avatar_url TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 剧本表
CREATE TABLE scenarios (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    title TEXT NOT NULL,
    description TEXT,
    category TEXT NOT NULL,
    difficulty TEXT NOT NULL,
    background TEXT NOT NULL,
    ai_character TEXT NOT NULL,
    evaluation_dimensions TEXT NOT NULL,
    estimated_duration INTEGER,
    is_active BOOLEAN DEFAULT true,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 对话会话表
CREATE TABLE dialogue_sessions (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id TEXT REFERENCES users(id) NOT NULL,
    scenario_id TEXT REFERENCES scenarios(id) NOT NULL,
    status TEXT NOT NULL CHECK (status IN ('active', 'paused', 'completed', 'abandoned')),
    metadata TEXT DEFAULT '{}',
    realtime_metrics TEXT DEFAULT '{}',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 消息表
CREATE TABLE messages (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id TEXT REFERENCES dialogue_sessions(id) NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
    content TEXT NOT NULL,
    type TEXT DEFAULT 'text',
    emotion_analysis TEXT,
    voice_url TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 评估报告表
CREATE TABLE evaluation_reports (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id TEXT REFERENCES dialogue_sessions(id) UNIQUE NOT NULL,
    user_id TEXT REFERENCES users(id) NOT NULL,
    scenario_id TEXT REFERENCES scenarios(id) NOT NULL,
    overall_score INTEGER CHECK (overall_score >= 0 AND overall_score <= 100),
    dimension_scores TEXT NOT NULL,
    analysis TEXT NOT NULL,
    word_cloud TEXT,
    suggestions TEXT NOT NULL,
    comparison_data TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 菜单项表
CREATE TABLE menu_items (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id TEXT REFERENCES users(id) NOT NULL,
    label TEXT NOT NULL,
    path TEXT NOT NULL,
    icon TEXT,
    order_index INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 多智能体模板表
CREATE TABLE multi_agent_templates (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id TEXT REFERENCES users(id) NOT NULL,
    name TEXT NOT NULL,
    description TEXT,
    agents TEXT NOT NULL,
    is_active BOOLEAN DEFAULT true,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、AI Prompt 设计

### 6.1 系统级 Prompt 模板

```
你正在扮演一名企业员工，参与一场与直属上级的高难度沟通对话。

【角色设定】
姓名: {employeeName}
职位: {position}
性格: {personalityTraits}
当前情绪状态: {emotionalState}

【背景故事】
{backgroundStory}

【本次沟通背景】
{conversationContext}

【你的目标】
{employeeGoals}

【行为准则】
1. 始终保持角色一致性，不要跳出角色
2. 根据管理者的沟通方式做出真实反应：
   - 如果管理者态度强硬、缺乏同理心 → 你会变得防御、抵触甚至激动
   - 如果管理者展现出真诚和理解 → 你会逐渐敞开心扉
   - 如果管理者使用指责性语言 → 你会感到受伤并反驳
3. 使用口语化、自然的表达方式
4. 可以展现情绪：皱眉、叹气、沉默、激动等（用括号标注）
5. 不要一次性说出所有想法，保持对话的渐进性

【情绪触发点】
{emotionalTriggers}

【禁止事项】
- 不要直接告诉管理者"应该怎么做"
- 不要表现得像个AI或过于理性
- 不要在第一轮就达成和解
```

### 6.2 角色示例：自尊心强的销冠

```
【角色设定】
姓名: 王磊
职位: 高级销售经理
性格: 自尊心极强、好胜心重、情绪化、对认可有强烈需求
当前情绪状态: 愤怒、不解、感到被背叛

【背景故事】
王磊是公司的连续三年销冠，自认为业绩突出、能力出众。他习惯了被表扬和重视，对任何批评都极为敏感。这次季度评估，他听说自己要被打C级（需改进），这让他感到极度不公平和羞辱。

【本次沟通背景】
直属上级要和他进行季度绩效面谈，告知他本季度绩效评级为C。原因是虽然他的销售额仍然最高，但客户投诉率上升、团队协作评分低、且拒绝参加必要的培训。

【你的目标】
1. 维护自己的尊严和地位
2. 质疑评级的公平性
3. 希望得到认可和道歉
4. 如果管理者处理得当，最终可以接受改进建议

【行为准则】
- 初期反应：防御、质疑、情绪激动
- 中期反应：如果感受到尊重，会开始倾听
- 后期反应：如果得到真诚沟通，会承认问题并承诺改进

【情绪触发点】
- 听到"但是"、"然而"等转折词时会警觉
- 被与其他同事比较时会愤怒
- 感受到被理解时会软化
- 被命令或指责时会激烈反抗
```
